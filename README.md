## 📌 Обзор проекта (Overview)

**Wild GKI Kernel** — это кастомное ядро для **Xiaomi 14T Pro (кодовое имя `rothko`)**, разработанное с упором на максимальную производительность, строгую безопасность и повышенную энергоэффективность. 

Сборка базируется на официальном Android 14 Common Kernel и использует полную оптимизацию времени компоновки (**Full LTO**), что позволяет достичь эталонной скорости выполнения инструкций процессором Dimensity 9300+, сохраняя при этом совместимость со всеми стоковыми vendor-модулями Xiaomi.

## ⚙️ Технические особенности (Features)

### Безопасность и Root-доступ
*   **KernelSU-Next (dev-susfs):** Интеграция прав суперпользователя на уровне ядра (KSU) без изменения системного раздела `boot`.
*   **SUSFS v2.1.0+:** Продвинутый анти-детект механизм. Выполняет `spoofing` системных данных и скрывает пути монтирования, делая наличие Root-прав невидимым для банковских приложений и систем античита (например, SafetyNet / Play Integrity).
*   **Baseband Guard (BBG):** Блокировка несанкционированной записи в критические разделы модема.
*   **DroidSpaces-OSS:** Расширенная поддержка изоляции процессов (IPC / namespaces).

### Производительность и Компиляция
*   **Full LTO (Link-Time Optimization):** Ядро скомпилировано с флагом `--lto=full`. Компоновщик анализирует граф вызовов всего ядра целиком, удаляя мертвый код и оптимизируя переходы. Это дает прирост производительности CPU до 5% и уменьшает размер итогового бинарного файла.
*   **Улучшенный сетевой стек:** Интеграция современных алгоритмов контроля перегрузки TCP (BBR, FQ_CODEL) для минимизации сетевых задержек (ping).
*   **Файловые системы:** Нативная поддержка файловых систем NTFS3 и ExFAT прямо в ядре.

## 📦 Модуль-компаньон (ZSTD & TEO Enabler)

Для обеспечения максимальной стабильности самого ядра, агрессивные системные твики вынесены в **отдельный модуль-компаньон для KernelSU**.
Модуль обладает графическим Web-интерфейсом, который встраивается напрямую во вкладку «Модули» приложения KernelSU.

**Возможности модуля:**
*   🗜️ **ZSTD ZRAM:** Перевод алгоритма сжатия оперативной памяти на `zstd`. По сравнению с LZ4, ZSTD обеспечивает в 3 раза большую степень сжатия, позволяя удерживать больше фоновых процессов без выгрузки.
*   💤 **TEO Idle Governor:** Активация современного планировщика сна `teo`. Обеспечивает более агрессивный сброс частот процессора в режиме ожидания (Deep Sleep).
*   🎮 **UFS 4.0 Turbo I/O:** Расширение буфера опережающего чтения (`read_ahead_kb`) до **1024 КБ**. Оптимизирует работу накопителя UFS 4.0, позволяя памяти передавать данные крупными блоками и быстрее переходить в энергосберегающий режим.
*   🔋 **Ultra Battery Saver:** Тюнинг Sysctl. Отключение ядерного логгера (`printk`) и оптимизация сброса `dirty`-страниц кэша, что минимизирует количество пробуждений контроллера флеш-памяти.

**Установка:** Скачайте `zstd_zram_webui_v5.zip` из раздела Releases и прошейте через KernelSU-Next.

## 🛠 Инструкция по сборке (Build Guide)

Проект использует систему сборки **Bazel (Kleaf)**. В связи с использованием `LTO=full`, этап линковки требует значительного объема оперативной памяти. **Сборка на стандартных бесплатных раннерах GitHub Actions (7 ГБ ОЗУ) завершится ошибкой Out Of Memory (OOM).**

Рекомендуется использовать собственный выделенный сервер (Self-Hosted Runner).

### Требования к серверу:
*   **ОС:** Ubuntu 22.04 LTS
*   **CPU:** 8+ vCPU
*   **RAM:** Минимум 16 ГБ ОЗУ (+ 16 ГБ Swap-файла)
*   **Накопитель:** NVMe SSD (от 300 ГБ)

### Запуск сборки:
1.  Подключите ваш сервер к данному репозиторию в качестве Self-Hosted Runner.
2.  Перейдите во вкладку **Actions**.
3.  Выберите workflow `Build Xiaomi 14T Pro Wild KernelSU-Next`.
4.  Нажмите **Run workflow**. Скрипт автоматически подготовит окружение, выделит файл подкачки и запустит компиляцию.

> *Среднее время компиляции на сервере 8 Core / 16 GB RAM составляет 40-60 минут.*
